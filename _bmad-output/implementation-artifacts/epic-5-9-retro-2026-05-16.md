## Successes

### 2. Architectural Breakthroughs That Set Patterns

- **Share-token write path** (Story 5.2): First reviewer write operation via scoped tokens without RBAC — principled authorization via layered checks (authenticated user → token lookup → reviewer match → share validity → vacancy scorecard existence). Reusable pattern.
- **Immutable audit hash chain** (Story 7.6): Cryptographic integrity using SHA-256 hash chain with `$transaction` for race condition prevention. Sequential numbering + hash provides tamper evidence.
- **Atomic hire-to-onboarding bridge** (Story 8.2): Starter creation + stage move as atomic operation with explicit rollback on partial failure.
- **Entity-scoped email templates** (Story 6.1): Separate `RecruitmentEmailTemplate` model instead of extending existing `EmailTemplate` — correct architectural decision for multi-tenant candidate-facing communications.

## Challenges

### 1. Story Spec Quality Inconsistency Across Epics

| Epic | Spec Quality | Detail Level |
|------|-------------|--------------|
| 5 | Excellent | Full RBAC matrices, anti-patterns, file structure, previous story intelligence, security checklists |
| 6 | Good | Dev notes with existing infrastructure references, but less RBAC detail in stories 6.2-6.8 |
| 7-9 | Minimal | Compact task lists, short dev notes, minimal architectural guidance |

**Root cause:** As the development workflow accelerated (batch creation + batch implementation for later epics), story specs were generated with less individual attention. The quality inversely correlated with batch size.

**Impact:** Higher reliance on dev agent's own judgment for architectural decisions in Epic 7-9. Code review found issues that richer specs might have prevented (PII in audit logs, atomicity gaps).

### 2. PII Leaks in Audit Logs (Recurring Pattern, Epic 7)

Two separate audit log calls included PII in metadata that should have been anonymous:
- `retention-purge` route logged `originalId` of a deleted candidate
- `erasure` route logged `candidateName`

**Root cause:** Standard dev mindset treats audit logs as "helpful debug context." GDPR compliance requires the opposite — audit logs must survive data deletion without containing PII. This adversarial thinking isn't automatic.

### 3. Atomicity Gaps in Multi-Step Operations (Epic 8)

The hire API created a Starter record within a Prisma transaction but called `executeStageMove` outside it. If the stage move failed, an orphaned Starter would exist.

**Root cause:** Prisma transactions don't natively wrap external function calls that do their own DB operations. The solution (try/catch with manual rollback) works but requires explicit architectural awareness.

### 4. Performance as Deferred Concern

- Vacancies API fetches all active candidates for SLA calculation
- Audit report API without index optimization for large datasets
- Rate-limit in-memory Map without eviction (pre-existing from Epic 3)

Accepted for current scale but creating accumulated technical debt.

## Key Insights

1. **Retrospective action items deliver measurable results** — The RBAC template investment from Epic 3+4 eliminated an entire bug category. This is the strongest evidence for the value of structured retrospectives.

2. **Code review severity evolves with team maturity** — Finding types shifted from RBAC permission bugs (Epic 3-4) to data integrity and compliance issues (Epic 5-9). The review focus must evolve correspondingly.

3. **Compliance code requires adversarial thinking** — GDPR-related code (retention, purge, export, erasure) requires thinking from the attacker/auditor perspective. Standard feature development mindset misses PII leaks in audit logs.

4. **Spec quality correlates with implementation quality** — Epic 5's rich story specs produced the cleanest implementations. Epic 7-9's minimal specs required more code review patches. The investment in spec writing pays for itself.

5. **25 stories in 5 epics successfully delivered** — The complete Recruitment Module is feature-complete: from vacancy creation to analytics dashboard, including full GDPR compliance, onboarding integration, and real-time collaboration.
